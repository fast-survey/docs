With the Email field you can obtain the email address of the user. We will verify the submitted address via a confirmation email. You have the possibility to specify a regex to constrain valid email addresses, e.g. `.*@gmail.com` to accept only Gmail addresses.

## Properties

- `title`: the title of the question
- `description`: a short optional description containing more details
- `regex`: the regex by which valid email addresses are constrained
- `hint`: a textual hint e.g. how a valid email address could look like

## Rules

- `title [string]`: `1 <= title.length <= 120`
- `description [string]`: `0 <= description.length <= 10000`
- `regex [string]`: valid format and `regex.length <= 1000`
- `hint [string]`: `0 <= hint.length <= 120`

## Example JSON Specification:

```json
{
    "survey_name": "email",
    "title": "Email Test",
    "description": "",
    "start": 1000000000,
    "end": 2000000000,
    "draft": false,
    "authentication": "open",
    "limit": 0,
    "fields": [
        {
            "type": "email",
            "title": "What is your email address?",
            "description": "",
            "regex": ".*",
            "hint": "Your email address ..."
        }
    ]
}
```

## Example of a Valid Submission

```json
{
    "1": "support@fastsurvey.io"
}
```
